# familyfit
🏠 FamilyFit：連結家人的溫暖尺碼庫
💡 開發初心 (Original Intention)
「愛是記得你穿幾號鞋，也記得你喜歡什麼顏色。」

我發現，在家庭生活中，我們卻常因為數據的「模糊」而錯失表達心意的機會。 每次出國想幫爸媽買雙好走的鞋，卻不確定他們的腳是 24.5 還是 25 號；想送太太一件衣服，卻怕買小了不合身、買大了不體貼。我們往往因為「不確定尺碼」而放棄了送禮的驚喜，或者需要在群組裡反覆確認，少了那份驚喜感。

FamilyFit 的誕生，就是為了解決這個溫暖的小煩惱。 我希望打造一個**「專屬於家庭的數據規格表」**，讓每一位家人的身體密碼（從身高體重到戒指圍度）都被妥善保存。當你想表達心意時，打開 App，所有的數據都在那裡，準確且安心。

✨ 核心功能與設計思維
為了讓長輩也能輕鬆使用，同時保有工程師對數據的堅持，我堅持了以下設計原則：

一人一表，規格化管理 (Spec Sheet Style)

摒棄雜亂的聊天記錄，採用類似機械規格表的「垂直卡片」設計。

詳細參數： 涵蓋身高、體重、褲碼/腰圍，甚至細緻到戒指圍度與佩戴手指。

標準化鞋碼： 解決各國鞋碼混亂問題，強制採用「標準(EU/US)+數值」的組合輸入。

隱私與便捷的平衡 (Privacy by Design)

分流驗證： 註冊與登入分離，確保帳號安全。

群組隔離機制： 採用「房號 + PIN 碼」的虛擬房間概念。只有輸入相同 4 位數 PIN 碼的家人才能進入同一個空間，既方便分享，又有足夠的隱私隔離。

安全限制： 強制 7 位數帳號密碼與 4 位數群組 PIN 碼，防止誤操作與簡單的暴力破解。

跨越語言與裝置 (Accessibility)

雙語介面： 內建中/英一鍵切換，無論是家中長輩或海外親戚都能無障礙使用。

免安裝 (PWA)： 利用瀏覽器技術，無需下載 App，掃描 QR Code 即可使用，並支援「加入主畫面」成為原生 App 般的體驗。

數據鮮度 (Freshness)

更新時間戳： 每個人的檔案都會自動記錄「最後更新日期」，確保你看到的尺碼是最新的，避免買到幾年前的舊尺寸。

🛠️ 技術棧
Frontend: 原生 HTML5 / CSS3 (Responsive Design)

Backend: Google Firebase (Firestore Database)

Deployment: GitHub Pages

Security: Google Cloud API Key Restrictions (HTTP Referrer)
